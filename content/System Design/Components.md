System design components:
- [[load balancing]]: distribute network/application traffic across a number of servers
- [[caching]]: store a copy of data in temporary storage and serve this cached data for future requests
- [[partitioning]]: divide a database into smaller sub-divisions
- [[replication]]: create and maintain multiple copies of data
- [[message queue]]: offload tasks from the main application threads
- [[batch processing]]: execute a series of jobs at once, rather than sequentially
